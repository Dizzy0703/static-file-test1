INSTALLING NODE


The most convenient way of installing Node.js depends on the host system you'll be working with. Below, the procedures for most major desktop Operating Systems are given. If you use Linux, keep in mind that it is considered dangerous and unnecessary to run sudo npm, as this can cause annoying package-management issues.


Windows NT

For Windows users, Node.js can be downloaded as a .msi Windows Installer file from the Official Node.js Website. It is recommended, unless you know what you're doing, that you download the LTS (Long-Term-Support) version, as it may contain fewer undocumented bugs.

If you don't have administrative privileges on your machine, a binary version in a zipped folder exists that may be downloaded and extracted to your user directory, and this directory added to your path variable.
If you downloaded the .msi version of Node, you only need to double-click the file (wherever you put it) and follow the on-screen prompts to complete installation. After this, Node should have added its own location to your PATH environment variable. If you downloaded the zipped binary, you will have to add the bin folder to your PATH manually.

To check that everything is working, open up a command prompt and type the following:

> node -v
> npm -v


macOS

For macOS users, Node.js can be downloaded as a .pkg Installer Package file, also from the Official Node.js Website. Again, the LTS version is recommended for most users.

A tarballed macOS binary exists for both Intel and ARM 64-bit processors, whose purpose is similar to the zipped folder provided for Windows users.
The installation procedure for Node is similar to any other Mac application. Once complete, Node should be recognised by the terminal, which you can check with the following:

$ node -v
$ npm -v


Linux

For Linux users, Node.js is available on the Official Node.js Website as prebuilt binaries for either x86_64 or the ARMv7 and ARMv8 instruction set architectures. All of these binaries are distributed as .xz-compressed tarballs, and should work with any distribution supporting the listed architectures (which would include Ubuntu, Debian, Linux Mint, etc.).

While Node.js can also be installed using a distribution's package manager, this is NOT recommended, as many package managers only list outdated versions of Node.js, and at any rate, setting up Node.js as a binary for individual accounts on a Linux machine is easier and, in some ways, safer, than installing it with superuser privileges.
Once you have the binaries for Node downloaded, extract them with the following:

$ tar -xf node-vxx.xx.x-linux-isa.tar.xz

Once extracted, add the directory they landed in to your path and check that they are recognised with the following:

$ node -v
$ npm -v
